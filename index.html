¿Qué es el DCL?

Un Lenguaje de Control de Datos (DCL por sus siglas en inglés: Data Control Language) es un lenguaje proporcionado por el sistema de gestión de base de datos que incluye una serie de comandos SQL que permiten al administrador controlar el acceso a los datos contenidos en la base de datos.

Comandos en DCL

Algunos ejemplos de comandos incluidos en el DCL son los siguientes:

  • GRANT: Permite dar permisos a uno o varios usuarios o roles para realizar tareas determinadas.
  • REVOKE: Permite eliminar permisos que previamente se han concedido con GRANT.
  • DENY: Prohíbe ciertos permisos a los usuarios.

Comandos en DCL

Las tareas sobre las que se pueden conceder o denegar permisos son las siguientes:

  • CONNECT
  • SELECT
  • INSERT
  • UPGRADE
  • DELETE
  • USAGE

	-- GRANT
	GRANT SELECT, INSERT, UPDATE, DELETE ON Employees to almir

	-- REVOKE
	REVOKE INSERT ON Employees to almir

	-- DENY
	DENY UPDATE ON Employees to almir

¿Qué es un rol en MySQL?

Un Rol actúa como plantillas que facilitan la asignación de permisos a un usuario de una base de datos. Se soportan los siguientes roles de usuario de base de datos: Lectura y escritura (usado de forma predeterminada), Solo lectura y Sólo escritura. Cada rol presenta un conjunto de privilegios predeterminados que se conceden al usuario de la base de datos al que se le asigne. Si lo desea, puede editar los conjuntos de privilegios correspondientes a los distintos roles.

Permisos Globales

Los permisos globales se aplican a todas las bases de datos de un servidor dado. Estos privilegios son almacenados en la tabla “mysql.user"

Elemplo


-- Asignar todos los pribilegios
GRANT all privileges on *.* to user1@'localhost' identified by 'user1' with grant option;

-- Remover todos los pribilegios
REVOKE all privileges on *.* FROM user1@'localhost' identified by 'user1';

Permisos Especificos

Existen 3 niveles para especificar permisos, estos pueden ser a nivel de base de datos, a nivel de tabla o a nivel de columna. Esto dependera de lo que se desea que el usuario tenga acceso.

Permisos de Base de Datos

Los permisos de base de datos se aplican a todos los objetos en una base de datos dada. Estos permisos se almacenan en las tablas “mysql.db”

Elemplo


-- Asignar todos los pribilegios a una sola base de datos
GRANT all privileges on cdcol.* to user1@'localhost' identified by 'user1' with grant option;

-- Remover todos los pribilegios de una solo base de datos
REVOKE all privileges on cdcol.* FROM user1@'localhost' identified by 'user1';

Permisos de Tabla

Elemplo


-- Asignar todos los pribilegios a una sola tabla de una base de datos
GRANT all privileges on cdcol.cds to user1@'localhost' identified by 'user1' with grant option;
		
-- Remover todos los pribilegios de una solo tabla
REVOKE all privileges on cdcol.cds FROM user1@'localhost' identified by 'user1';

Los permisos de tabla se aplican a todas las columnas en una tabla dada. Estos permisos se almacenan en la tabla "mysql.tables_priv"

Permisos de Tabla

Los permisos de tabla se aplican a todas las columnas en una tabla dada. Estos permisos se almacenan en la tabla "mysql.tables_priv"

Elemplo


-- Asignar todos los pribilegios a ciertas columnas de una tabla
GRANT SELECT(titel,interpret) on cdcol.cds to user1@'localhost' identified by 'user1' with grant option;
		
-- Remover todos los pribilegios de las columnas de la tabla
REVOKE SELECT(titel,interpret) on cdcol.cds FROM user1@'localhost' identified by 'user1';
